# SPECIMEN OF THOUGHT — Codex 개발 인수인계서

> 생각의 표본: 현대미술의 질문으로 철학을 해석하는 모바일 우선 반응형 웹사이트

## 0. Codex에 바로 전달할 실행 지시

이 문서를 프로젝트의 제품·콘텐츠 명세서로 사용한다. 먼저 저장소 구조와 기존 파일을 확인한 다음, 별도 프레임워크가 없다면 HTML, CSS, JavaScript만 사용하는 정적 단일 페이지 웹사이트로 구현한다. 모바일 390px 화면을 우선 설계하고 데스크톱까지 반응형으로 확장한다.

초기 버전에서는 AI API, 로그인, 데이터베이스를 사용하지 않는다. 철학적 정확성을 위해 이 문서에 정의된 검증형 선택지와 분기 대화만 사용한다. 사용자 선택은 `localStorage`에 저장한다. 외부 작품 이미지를 복제하지 말고 CSS, 자체 제작 도형, 추상적 실루엣으로 ‘유리·액체·보존·관찰’의 경험을 구현한다.

개발 중 임의로 콘텐츠 의미를 바꾸거나 데이미언 허스트 및 철학자의 실제 발언처럼 보이게 만들지 않는다. 모든 대화는 ‘사상과 작업 세계를 바탕으로 재구성한 가상 대화’임을 표시한다.

### 우선 구현할 결과물

1. 오프닝 및 프로젝트 안내
2. 최초 입장 선택
3. 철학자 3명 선택
4. 철학자별 분기 대화
5. 세 철학자 관점 비교 `CUBIST VIEW`
6. 최초·최종 판단 비교
7. 사용자별 생각 표본 카드 생성
8. 결과 저장 또는 공유 가능한 카드 UI
9. 모바일·태블릿·데스크톱 반응형
10. 접근성 및 키보드 조작 지원

## 1. 프로젝트 정의

### 프로젝트명

**SPECIMEN OF THOUGHT**  
국문명: **생각의 표본**

### 한 줄 정의

현대미술가의 도발적인 질문과 고대 철학자의 관점을 연결하여, 사용자가 죽음·실재·존재에 관한 자신의 판단 변화를 하나의 ‘생각 표본’으로 남기는 대화형 웹사이트.

### 핵심 문장

> 죽음을 전시하는 현대미술의 질문과 삶을 설명한 철학의 답변이 만나는 디지털 사고 실험실.

### 공통 탐구 질문

> 우리는 죽음을 직접 보아야 삶을 이해할 수 있는가?

### 프로젝트의 차별점

- 철학자가 정답을 알려주는 지식형 챗봇이 아니다.
- 현대미술의 문제의식으로 철학자의 사상을 시험한다.
- `ARTIST A`, `PHILOSOPHER`, `YOU`의 삼자 대화 구조를 사용한다.
- 사용자의 최초 판단과 최종 판단 사이 변화를 결과로 보여준다.
- 결과를 일반 채팅 기록이 아니라 ‘생각 표본’으로 보존한다.

## 2. 현대미술 관점과 표현 원칙

### ARTIST A

`ARTIST A`는 데이미언 허스트 본인을 복제한 캐릭터가 아니다. 허스트의 작품에서 반복되는 다음 문제의식을 바탕으로 질문하는 가상의 현대미술가다.

- 삶과 죽음
- 보존과 소멸
- 아름다움과 혐오
- 과학과 믿음
- 욕망과 소비
- 생명과 예술 재료의 윤리

`A`는 `Artist`, `Ask`, `Another perspective`를 동시에 의미한다.

### 반드시 표시할 면책 문구

> ARTIST A의 질문은 데이미언 허스트의 작품 세계에서 발견되는 문제의식을 바탕으로 재구성한 가상 대화입니다. 철학자의 답변 역시 주요 저술과 사상에 근거한 교육 목적의 재구성이며 실제 발언이 아닙니다.

### 피카소·입체주의의 사용

피카소를 등장인물이나 장식 스타일로 직접 사용하지 않는다. 대신 하나의 대상을 여러 시점으로 분해해 보는 입체주의의 원리를 `CUBIST VIEW`라는 비교 인터페이스에 적용한다.

## 3. 목표 사용자와 사용 상황

### 주요 사용자

- 철학을 어렵게 느끼지만 현대미술과 시각 경험에 관심 있는 청소년·청년
- 학교 발표나 전시에서 QR 코드로 3~7분 안에 체험하는 관람자
- 철학 개념을 암기보다 질문과 선택을 통해 이해하려는 사용자

### 목표 경험

사용자는 정답을 맞히는 것이 아니라 다음을 경험해야 한다.

1. 낯설고 불편한 질문과 마주한다.
2. 자신의 최초 입장을 선택한다.
3. 한 철학자의 관점을 따라 생각한다.
4. ARTIST A의 반문을 통해 자신의 판단을 의심한다.
5. 다른 철학자의 관점과 비교한다.
6. 달라지거나 유지된 판단을 표본으로 남긴다.

## 4. 기술 범위

### 권장 기술

- HTML5
- CSS3: custom properties, grid, flex, media queries, reduced-motion 대응
- Vanilla JavaScript ES6+
- 콘텐츠 데이터: 별도 `data.js` 또는 `dialogues.json`
- 상태 저장: `localStorage`
- 배포: Netlify 또는 GitHub Pages

### 권장 파일 구조

```text
/
├─ index.html
├─ styles.css
├─ app.js
├─ data/
│  └─ dialogues.js
├─ assets/
│  ├─ icons/
│  └─ textures/
└─ README.md
```

빌드 도구가 없는 저장소라면 의존성을 추가하지 않는다. 이미 적절한 프레임워크가 있는 경우 기존 구조를 유지하되, 정적 배포가 가능해야 한다.

### 초기 버전 제외 범위

- AI 자유 대화 API
- 회원가입과 로그인
- 서버 데이터베이스
- 실제 데이미언 허스트 작품 이미지
- 사용자의 민감정보 수집
- 복잡한 3D/WebGL

## 6. 화면별 와이어프레임 명세

### 00. 오프닝

**목적:** 세계관 진입.

**화면 요소**

- 거의 검은 배경
- 중앙의 추상적인 유리 수조 프레임
- 물속에 정지한 비구상적 실루엣
- 천천히 나타나는 문장

**카피**

> 죽음은 보이지 않을 때 사라지는가?  
> 아니면 더 강하게 상상되는가?

버튼: `표본실에 입장하기`

### 01. 프로젝트 안내

**카피**

> 이곳에는 정답이 없습니다.  
> 예술가의 질문, 철학자의 관점, 그리고 당신의 판단만 남습니다.

짧은 면책 문구와 `약 5분 소요` 표시.

버튼: `사고 실험 시작`

### 02. 최초 판단

**질문**

> 우리는 죽음을 직접 보아야 삶을 이해할 수 있다고 생각합니까?

**선택지 ID**

- `initial_yes`: 그렇다
- `initial_no`: 그렇지 않다
- `initial_unsure`: 아직 판단하기 어렵다

선택 시 `state.initialPosition`에 저장한다.

### 03. 철학자 선택

연대나 어려운 개념보다 질문을 앞세운다.

| ID | 화면 질문 | 철학자 | 보조 개념 |
|---|---|---|---|
| epicurus | 죽음은 왜 두려운가? | 에피쿠로스 | 죽음과 평온 |
| plato | 눈앞에 있는 것은 진짜인가? | 플라톤 | 실재와 이미지 |
| aristotle | 형태만 남아도 같은 존재인가? | 아리스토텔레스 | 질료·형상·생명 |

### 04. 대화 화면

**상단**

- 뒤로 가기
- 철학자 이름
- 진행률

**본문**

- `ARTIST A`: 붉은 라벨, 왼쪽 정렬
- `PHILOSOPHER`: 청록 라벨, 왼쪽 정렬
- `YOU`: 라임 라벨, 오른쪽 정렬
- 한 번에 한 발화 또는 짧은 발화 묶음만 표시

**하단**

- 48px 이상 높이의 선택 버튼 3개
- 답변 선택 후 다음 대화가 부드럽게 나타남
- 자동 스크롤하되 사용자가 위를 읽고 있을 때 강제로 끌어내리지 않음

### 05. CUBIST VIEW

**제목**

> 같은 대상을 세 개의 시선으로 본다면

**요약 카드**

- 에피쿠로스: 죽음의 공포는 현재의 상상에서 생긴다.
- 플라톤: 눈앞의 사체는 죽음의 본질 자체가 아니다.
- 아리스토텔레스: 형태는 남아도 생명 활동의 원리는 사라졌다.

모바일에서는 좌우 스와이프 또는 탭 전환, 데스크톱에서는 비대칭 3분할 그리드를 사용한다. 각 카드는 시각적으로 분리되지만 같은 중앙 대상의 일부를 공유하는 것처럼 보이게 한다.

### 06. 최종 판단

최초 질문을 동일하게 다시 제시한다.

**선택지 ID**

- `final_yes`: 필요하다
- `final_no`: 필요하지 않다
- `final_context`: 경우에 따라 다르다

### 07. 판단 변화

초기 판단 → 최종 판단을 선과 두 개의 표식으로 시각화한다.

예시:

```text
그렇지 않다  ─────────→  경우에 따라 다르다
최초 판단                  최종 판단
```

정답·점수·우열을 표시하지 않는다.

### 08. 생각 표본 카드

**필수 항목**

- 자동 증가 또는 랜덤 표본 번호
- 선택 철학자
- 최초 판단
- 최종 판단
- 핵심 표본 문장
- 생성 날짜
- `가상 대화 및 사상 기반 재구성` 라벨

**버튼**

- `표본 저장하기`
- `다른 철학자와 대화하기`
- `처음부터 다시 시작`

이미지 저장 기능이 어렵다면 첫 버전에서는 카드 영역을 인쇄·스크린샷하기 좋게 구현한다. 가능하면 `html-to-image` 같은 외부 의존성 없이 Web Share API 또는 인쇄 CSS를 우선 고려한다.

## 7. 상태 모델

```js
const initialState = {
  version: 1,
  currentScreen: "opening",
  initialPosition: null,
  selectedPhilosopher: null,
  dialogueStep: 0,
  answers: {
    epicurus: [],
    plato: [],
    aristotle: []
  },
  completedPhilosophers: [],
  finalPosition: null,
  specimenId: null,
  createdAt: null
};
```

### 저장 키

`specimen-of-thought:v1`

### 상태 규칙

- 선택할 때마다 저장한다.
- `다른 철학자와 대화하기`는 초기·최종 판단과 기존 표본을 보존하고 철학자 선택 화면으로 이동한다.
- `처음부터 다시 시작`은 명확한 확인 후 해당 키만 삭제한다.
- 저장된 버전이 다르면 안전하게 초기화한다.

## 8. 대화 데이터 구조 예시

실제 구현 시 아래와 유사한 데이터 기반 렌더링을 사용한다. 대화문을 HTML에 중복 하드코딩하지 않는다.

```js
const dialogues = {
  epicurus: {
    name: "에피쿠로스",
    theme: "죽음은 왜 두려운가?",
    source: "에피쿠로스, 「메노이케우스에게 보내는 편지」",
    nodes: {
      intro: {
        lines: [
          { speaker: "artist", text: "에피쿠로스, 당신은 죽음이 우리에게 아무것도 아니라고 했습니다. 그런데 사람들은 보존된 죽음 앞에서 왜 본능적으로 뒤로 물러납니까?" },
          { speaker: "philosopher", text: "그들이 두려워하는 것은 눈앞의 죽은 존재가 아닐 것입니다. 그 죽음에 비친 자신의 미래를 현재의 고통처럼 상상하고 있는 것이지요." }
        ],
        question: "당신은 무엇을 두려워한다고 생각합니까?",
        options: [
          { id: "nothing_remains", label: "죽은 뒤 아무것도 남지 않는 것", next: "e_nothing" },
          { id: "dying_pain", label: "죽어 가는 과정의 고통", next: "e_pain" },
          { id: "separation", label: "사랑하는 사람들과 헤어지는 것", next: "e_separation" }
        ]
      }
    }
  }
};
```

각 노드는 `lines`, `question`, `options`, `next`를 기본으로 하고 마지막 노드는 `resultKey`를 가진다.

## 9. 철학자 1 — 에피쿠로스 전체 대화

### 사상적 근거

에피쿠로스는 우리가 살아 있는 동안 죽음은 아직 존재하지 않고, 죽음이 찾아왔을 때는 감각하는 우리가 더 이상 존재하지 않으므로 죽음 그 자체는 우리에게 고통이 될 수 없다고 보았다.

### 도입

**ARTIST A**  
에피쿠로스, 당신은 죽음이 우리에게 아무것도 아니라고 했습니다. 그런데 사람들은 보존된 죽음 앞에서 왜 본능적으로 뒤로 물러납니까?

**에피쿠로스**  
그들이 두려워하는 것은 눈앞의 죽은 존재가 아닐 것입니다. 그 죽음에 비친 자신의 미래를 현재의 고통처럼 상상하고 있는 것이지요.

### 선택 1: 당신은 무엇을 두려워하는가?

#### E1-A. 죽은 뒤 아무것도 남지 않는 것

**에피쿠로스**  
아무것도 느낄 수 없는 상태를 지금의 감각으로 상상하기 때문에 두려운 것입니다. 존재하지 않는 동안에는 상실을 경험할 주체도 없습니다.

**ARTIST A**  
하지만 인간은 죽은 뒤에도 자신의 몸과 이름이 어떻게 남을지 끊임없이 고민합니다. 느낄 수 없는 미래가 현재의 삶을 지배하고 있는 셈 아닙니까?

#### E1-B. 죽어 가는 과정의 고통

**에피쿠로스**  
그렇다면 두려움의 대상은 죽음이 아니라 고통입니다. 고통은 완화하거나 대비할 수 있지만, 죽음을 무한한 공포로 상상하면 현재의 평온까지 잃게 됩니다.

**ARTIST A**  
예술은 때로 그 고통을 가리지 않고 확대해 보여줍니다. 고통을 직시하는 행위가 평온을 깨뜨리는 것일까요, 막연한 공포를 구체화하는 것일까요?

#### E1-C. 사랑하는 사람들과 헤어지는 것

**에피쿠로스**  
그것은 죽음 자체보다 관계의 상실에 대한 슬픔입니다. 죽음을 피할 수 없다는 사실은 바뀌지 않지만, 지금 함께하는 시간을 어떻게 살아갈지는 선택할 수 있습니다.

**ARTIST A**  
그렇다면 죽음을 보여주는 작품은 이별의 고통을 이용하는 것입니까, 남아 있는 관계의 가치를 깨닫게 하는 것입니까?

### 공통 질문 2

**ARTIST A**  
죽음을 눈앞에 전시하는 일은 막연한 공포를 줄이는 행위입니까, 아니면 공포를 볼거리로 만드는 행위입니까?

#### E2-A. 죽음을 직시하게 하므로 의미가 있다

**에피쿠로스**  
그 경험이 현재의 삶을 더 평온하고 충실하게 만든다면 의미가 있을 것입니다. 그러나 계속 불안에 머물게 한다면 죽음에 대한 집착을 강화할 뿐입니다.

#### E2-B. 죽음을 자극적인 볼거리로 소비한다

**에피쿠로스**  
타인의 죽음을 자신의 자극을 위해 소비한다면, 죽음에 대한 이해보다 욕망을 키우는 일이 될 수 있습니다. 중요한 것은 무엇을 보았는지가 아니라, 본 이후 어떻게 살아가는가입니다.

#### E2-C. 의미와 소비가 동시에 존재할 수 있다

**에피쿠로스**  
인간의 경험은 하나의 의도로만 이루어지지 않습니다. 충격에 끌려 다가갔다가 삶을 성찰할 수도 있습니다. 다만 충격 자체를 진실로 착각하지는 말아야 합니다.

### 에피쿠로스 결과 문장

- `E-03 AMBIVALENCE`: 죽음의 이미지는 성찰을 만들 수도, 공포를 소비하게 할 수도 있다. 의미는 이미지보다 그것을 본 이후의 태도에서 결정된다.

## 10. 철학자 2 — 플라톤 전체 대화

### 사상적 근거

플라톤은 감각으로 접하는 현실이 완전한 진리 자체가 아니며, 사물의 본질은 이성적 사유를 통해 이해할 수 있다고 보았다. 강렬한 이미지나 감정이 진리를 자동으로 보장하지 않는다는 문제를 중심으로 구성한다.

### 도입

**ARTIST A**  
플라톤, 이것은 실제로 죽은 생명체입니다. 그림이나 조각처럼 외형을 모방한 것도 아니지요. 그렇다면 관객은 지금 진짜 죽음을 보고 있는 것입니까?

**플라톤**  
관객이 보는 것은 죽음의 본질이 아니라 죽은 육체의 외형일 것입니다. 실제 사체를 제시했다고 해서 죽음이라는 진리를 그대로 보여주었다고 할 수는 없습니다.

**ARTIST A**  
그렇다면 실제 사체조차 죽음의 이미지일 뿐이라는 뜻입니까?

**플라톤**  
눈앞의 대상은 실제 육체이지만, 관객이 그것을 통해 떠올리는 죽음의 의미는 대상 그 자체와 같지 않습니다. 보이는 것과 그것이 의미하는 것을 구분해야 합니다.

### 선택 1: 유리 상자 속 대상은 무엇인가?

#### P1-A. 실제 죽음 그 자체

**플라톤**  
죽음의 결과를 보고 있다고 해서 죽음의 본질까지 보고 있다고 할 수 있을까요? 당신은 멈춘 육체를 보고 있지만, 생명이 사라진다는 것이 무엇인지는 눈으로 직접 확인할 수 없습니다.

**ARTIST A**  
관객은 가짜 이미지보다 실제 사체 앞에서 훨씬 강한 감정을 느낍니다. 감정의 강도가 대상의 진실성을 증명하지는 않습니까?

**플라톤**  
강렬한 감정은 경험의 힘을 보여주지만, 그것이 곧 진리의 증거는 아닙니다. 그림자도 충분히 크고 두려울 수 있지만, 그림자가 실체가 되는 것은 아닙니다.

#### P1-B. 죽음을 보여주는 이미지

**플라톤**  
사체는 실제이지만 작품 속에서 연출되고 배치된 순간 하나의 이미지로 작동할 수 있습니다.

**ARTIST A**  
실제 존재를 전시장에 옮겨놓는 순간 이미지가 된다면, 미술관은 현실을 진실하게 보여주는 장소입니까, 현실을 새롭게 꾸미는 장소입니까?

**플라톤**  
미술관은 대상을 선택하고 분리하며 특정한 방식으로 보게 합니다. 그곳에서 보는 현실은 이미 해석된 현실에 가깝습니다.

#### P1-C. 실제 사체이자 죽음의 상징

**플라톤**  
실제 육체가 존재하는 것과 그것이 죽음 전체를 대표하는 것은 서로 다른 문제입니다.

**ARTIST A**  
그렇다면 이 작품은 현실과 이미지 사이에 있는 것일까요?

**플라톤**  
그 경계에서 중요한 것은 관객이 보이는 외형에 머무르는지, 그 외형을 넘어 본질을 질문하는지에 있습니다.

### 공통 질문 2

**ARTIST A**  
관객이 이 작품 앞에서 마주하는 것은 죽은 생명체입니까, 아니면 언젠가 죽을 자기 자신의 모습입니까?

#### P2-A. 작품 속 실제 대상

**플라톤**  
감각은 우선 눈앞의 대상을 향합니다. 그러나 보이는 외형만 관찰한다면 작품이 던지는 질문보다 표면적인 충격에 머물 수 있습니다.

#### P2-B. 대상에 비친 나 자신

**플라톤**  
눈앞의 대상은 자신의 유한성을 생각하게 만드는 계기가 됩니다. 그러나 자기 감정을 대상의 의미 전부라고 단정해서는 안 됩니다. 성찰 역시 하나의 해석이기 때문입니다.

#### P2-C. 대상과 나 자신을 동시에 본다

**플라톤**  
감각으로는 대상을 보고, 사유를 통해 자신의 존재를 돌아보는 것입니다. 작품은 보이는 것 너머를 질문하게 하는 매개가 될 수 있습니다.

**ARTIST A**  
예술은 진리를 그대로 보여줄 수는 없지만, 진리를 향해 생각하게 만들 수는 있습니까?

**플라톤**  
예술이 외형의 매혹에 머물지 않고 영혼을 사유로 이끈다면, 진리를 향한 질문의 출발점은 될 수 있을 것입니다.

### 선택 3: AI 이미지 확장

**ARTIST A**  
실제 생명체의 사진, 사실적인 조형물, AI로 생성한 존재하지 않는 생명체가 똑같이 진짜처럼 보인다면 무엇을 기준으로 진실을 판단해야 합니까?

- 실제 대상을 촬영했는지가 중요하다.
- 이미지가 전달하는 의미가 중요하다.
- 제작 과정과 의미를 함께 확인해야 한다.

**응답 A**  
실제 대상을 근거로 했다는 사실은 중요하지만 사진도 대상을 선택하고 자르고 배열합니다. 현실을 기록한 이미지가 현실 전체를 보여주지는 않습니다.

**응답 B**  
존재하지 않는 이미지도 진실한 생각을 전달할 수 있습니다. 그러나 설득력 있는 이미지가 거짓을 진실처럼 보이게 할 위험도 있습니다.

**응답 C**  
보이는 결과뿐 아니라 무엇을 근거로 어떻게 만들어졌는지 질문하는 것은 감각적 인상에 머무르지 않고 이미지의 구조를 살펴보는 태도입니다.

### 플라톤 결과 문장

- `P-01 APPEARANCE`: 실제 대상을 사용한 작품은 현실의 물질적 흔적을 제시한다. 그러나 보이는 것이 곧 본질 전체인지는 계속 질문해야 한다.
- `P-02 REPRESENTATION`: 전시장에 놓인 현실은 이미 선택되고 분리된 현실이다. 작품은 진실 자체보다 누군가의 시선으로 구성된 이미지를 보여준다.
- `P-03 BEYOND THE VISIBLE`: 예술은 진리를 그대로 담을 수 없지만, 보이는 것 너머에 무엇이 있는지 질문하게 만들 수 있다.

## 11. 철학자 3 — 아리스토텔레스 전체 대화

### 사상적 근거

아리스토텔레스는 개별 사물이 질료와 형상의 결합으로 이루어진다고 보았다. 살아 있는 존재의 형상은 단순한 외곽이 아니라 감각·운동·생명 활동을 가능하게 하는 원리까지 포함한다.

### 도입

**ARTIST A**  
아리스토텔레스, 눈앞의 생명체는 죽기 전과 거의 같은 형태로 보존되어 있습니다. 그렇다면 이것은 여전히 같은 존재입니까?

**아리스토텔레스**  
같은 물질과 외형이 남아 있다는 점에서는 연속성이 있습니다. 하지만 생명체를 그 생명체이게 했던 감각과 운동, 생명 활동은 더 이상 실현되지 않습니다.

**ARTIST A**  
형태가 유지되어도 존재의 본질은 달라졌다는 뜻입니까?

**아리스토텔레스**  
살아 있는 몸은 특정한 모양의 물질이 아니라 생명을 실현할 수 있는 몸입니다. 그 능력을 잃었다면 외형은 남아 있어도 존재 방식은 달라진 것입니다.

### 선택 1: 수조 속 대상은 무엇인가?

#### A1-A. 죽었지만 여전히 같은 생명체

**아리스토텔레스**  
과거에 어떤 생명체였는지를 확인할 수 있다는 점에서는 같은 이름을 사용할 수 있습니다. 그러나 살아 있을 때와 죽은 뒤의 존재 방식까지 동일하다고 보기는 어렵습니다.

**ARTIST A**  
같은 이름을 사용하는 이유는 대상이 그대로이기 때문입니까, 과거의 기억을 붙잡고 있기 때문입니까?

**아리스토텔레스**  
이름은 연속성을 표현할 수 있지만 대상의 상태까지 보존하지는 못합니다.

#### A1-B. 생명체의 형태만 남은 물질

**아리스토텔레스**  
살아 있던 존재의 질료와 외형은 남아 있지만 생명 활동은 사라졌습니다.

**ARTIST A**  
그렇다면 나는 생명을 보존한 것이 아니라 생명을 잃은 물질을 보존한 것에 불과합니까?

**아리스토텔레스**  
생명 자체가 아니라 생명이 존재했다는 물질적 흔적을 보존했다고 할 수 있습니다.

#### A1-C. 이전의 존재와 새로운 예술작품이 겹쳐 있다

**아리스토텔레스**  
같은 물질이라도 놓인 목적과 관계가 바뀌면 다르게 이해될 수 있습니다. 살아 있을 때는 생명체였지만 전시장에서는 관찰과 사유의 대상으로 기능합니다.

### 공통 질문 2

**ARTIST A**  
생명이 빠져나간 자리에 예술적 의미를 채워 넣는 것은 창조입니까, 죽음을 이용하는 일입니까?

#### A2-A. 새로운 의미를 만드는 창조

**아리스토텔레스**  
예술은 재료에 일정한 형식과 목적을 부여합니다. 그러나 새로운 의미가 만들어졌다는 사실만으로 그 과정이 정당화되는 것은 아닙니다.

#### A2-B. 죽은 존재를 재료로 이용하는 행위

**아리스토텔레스**  
대상을 어떤 목적으로 사용하는지는 행위의 성격을 판단하는 중요한 기준입니다. 좋은 목적만으로 모든 수단이 정당해지지는 않습니다.

#### A2-C. 창조와 이용이 동시에 존재한다

**아리스토텔레스**  
작품은 새로운 사유를 만들면서 동시에 한 존재를 재료화할 수 있습니다. 예술적 가치와 윤리적 문제를 함께 검토해야 합니다.

### 선택 3: 동일성 확장

**ARTIST A**  
보존 과정에서 피부와 장기, 액체와 구조물을 계속 교체한다면 어느 순간부터 이것은 원래의 존재가 아니게 됩니까?

- 원래 물질이 남아 있어야 같은 존재다.
- 형태가 유지되면 같은 존재다.
- 기억과 이름이 이어지면 같은 존재다.

**응답 A**  
질료의 연속성을 중시하는 판단입니다. 그러나 생명체의 몸도 살아 있는 동안 끊임없이 변화하므로 물질만으로 동일성을 완전히 설명하기는 어렵습니다.

**응답 B**  
형태는 대상을 인식하게 하는 중요한 조건입니다. 하지만 살아 있는 존재의 형상은 외곽선이 아니라 생명 활동의 원리까지 포함합니다.

**응답 C**  
대상을 기억하고 해석하는 공동체의 관계를 강조합니다. 다만 기억 속 존재와 현재의 물질적 존재는 구분할 필요가 있습니다.

### 아리스토텔레스 결과 문장

- `A-01 CONTINUITY`: 물질과 형태의 연속성은 과거의 존재를 현재에 남긴다. 그러나 같은 이름이 같은 존재 방식을 의미하지는 않는다.
- `A-02 REMAINS`: 형태는 남아 있지만 생명 활동은 사라졌다. 보존된 것은 생명이 아니라 생명이 존재했다는 물질적 흔적이다.
- `A-03 TRANSFORMATION`: 같은 물질은 목적과 관계가 달라지면서 새로운 존재 방식으로 전환된다. 과거의 생명과 현재의 작품은 한 대상 안에 겹쳐 있다.

---

## 12. 최종 결과 매핑

최종 입장과 선택 철학자에 따라 해당 철학자의 결과 문장 중 하나를 선택한다. 단순히 마지막 버튼만 보지 말고 철학자 대화의 핵심 선택도 반영한다.

### 권장 규칙

- 최종 `yes`: 각 철학자의 01 계열
- 최종 `no`: 각 철학자의 02 계열
- 최종 `context`: 각 철학자의 03 계열

향후에는 대화 선택에 가중치를 부여할 수 있으나 MVP에서는 위 규칙으로 명확하게 구현한다.

### 판단 변화 문구

- 초기와 최종 방향이 달라짐: `당신의 판단은 대화를 지나며 이동했습니다.`
- 유보에서 입장 선택: `불확실한 질문 속에서 하나의 관점을 선택했습니다.`
- 입장 유지: `판단은 유지되었지만, 그 판단을 설명하는 이유가 확장되었습니다.`
- 최종 맥락 의존: `하나의 정답보다 상황과 관점의 차이를 남겼습니다.`

## 13. 비주얼 디자인 시스템

### 핵심 방향

‘허스트처럼 보이는 웹사이트’를 만들지 않는다. 보존·분류·관찰이라는 작업 원리를 인터페이스로 번역한다.

### 컬러 토큰

```css
:root {
  --color-bg: #080b0d;
  --color-surface: #10171a;
  --color-glass: rgba(169, 214, 210, 0.12);
  --color-glass-line: rgba(169, 214, 210, 0.42);
  --color-text: #ecefed;
  --color-muted: #9da8a6;
  --color-artist: #e84a3c;
  --color-philosopher: #a9d6d2;
  --color-user: #c7f464;
  --color-focus: #f4d35e;
}
```

### 타이포그래피

- 한글 본문: 시스템 산세리프 우선. 외부 폰트를 쓰는 경우 로딩 실패 대비 폴백 필수.
- 영문 라벨: 좁고 정밀한 산세리프 또는 모노스페이스.
- 모바일 본문 최소 16px, 줄간격 1.65 이상.
- 긴 철학 대화는 한 화면에 2~4줄 단위로 끊는다.

### 그래픽 모티프

- 얇은 유리 프레임
- 액체처럼 천천히 이동하는 그라데이션
- 표본 번호와 분류 라벨
- 겹친 반투명 면
- 확대·초점·관찰 표시
- 세 관점을 분리하는 비대칭 절단면

### 모션

- 기본 전환 300~500ms
- 수조 배경은 12~20초의 매우 느린 움직임
- 선택 시 짧은 굴절 또는 경계선 발광
- `prefers-reduced-motion: reduce`에서는 모든 장식 애니메이션 제거
- 깜박임, 급격한 확대, 반복적인 강한 패럴랙스 금지

### 반응형 기준

- 모바일 기준: 390px
- 작은 모바일: 320px 이상 지원
- 태블릿: 768px
- 데스크톱: 1100px 최대 콘텐츠 폭
- 본문 좌우 여백: 모바일 20~24px
- 버튼 최소 높이: 48px
- 터치 타깃 간격: 최소 8px

## 14. 접근성과 UX 기준

- 모든 버튼은 `button` 요소를 사용한다.
- 키보드만으로 전체 경험을 완료할 수 있어야 한다.
- 포커스 링을 제거하지 않는다.
- 색상만으로 화자를 구분하지 않고 라벨과 정렬을 함께 사용한다.
- 본문 대비는 WCAG AA를 목표로 한다.
- 장식용 실루엣은 `aria-hidden="true"` 처리한다.
- 화면 전환 시 제목에 포커스를 이동하고 스크린리더에 현재 단계가 전달되도록 한다.
- 대화가 추가될 때 적절한 `aria-live`를 사용하되 과도한 재낭독을 막는다.
- 저장 실패나 localStorage 비활성화 시에도 세션 중 체험은 가능해야 한다.

## 15. 콘텐츠·윤리 원칙

- 실제 발언과 재구성 대화를 혼동시키지 않는다.
- 실제 동물 사체나 충격적인 죽음 사진을 사용하지 않는다.
- 상어의 직접 복제보다 비구상적인 생명 흔적을 사용한다.
- 철학자 답변을 절대적 정답으로 표시하지 않는다.
- 죽음에 관한 콘텐츠라는 점을 시작 전에 안내한다.
- 어린 사용자를 고려해 선정적·혐오적 묘사를 피한다.
- 출처는 철학자별 대화 마지막 또는 정보 모달에서 확인 가능하게 한다.
